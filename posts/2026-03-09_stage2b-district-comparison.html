Stage 2b: District Validation — 5-Experiment Comparison

Project A · Progress Report — 2026-03-09

Project A
GDP Mapping
Project A — NDVI model achieves district correlation r=0.75 (2023), a 75% improvement over the NIR baseline (r=0.43). Alpha Earth embeddings catastrophically fail at district level (r=-0.29). Pixel AUC does not predict district-level economic correlation.
Author

Andrea Salem

Published

March 9, 2026

1 Context

This report extends Stage 2 with district-level validation for all five models trained in Stage 1c. The question: which spectral configuration produces the most economically meaningful building density map?

Stage 1c showed that pixel-level AUC favors the simple NIR baseline (0.976) over all richer spectral inputs. But Stage 2 demonstrated that pixel AUC is a poor proxy for the downstream metric that matters — correlation with district-level economic activity. This report resolves the question by running full-country inference and district aggregation for each model.

2 Method

For each of the 5 models:

  1. Inference: Run sliding-window prediction (128×128 patches, stride 64) over all 20 Rwanda chunks
  2. Aggregation: Count predicted building pixels per GADM district (30 districts)
  3. Correlation: Pearson r against Rwanda Establishment Census (2020 + 2023)

All models use threshold 0.50 for binary classification during district aggregation (except NIR baseline at 0.90, as established in Stage 1b). Inference and aggregation scripts: src/inference/01_predict_rwanda.py, src/inference/02_aggregate_districts.py.

3.2 Headline Result

NDVI achieves r = 0.750 (2023) — a 75% improvement over the NIR baseline’s r = 0.432. Adding a single NDVI band transforms a noisy economic proxy into a meaningful one.

This result confirms the Stage 2 error analysis prediction: NDVI separates false positives (bare soil, roads) from true buildings. With NDVI as an input, the model learns to suppress detections in non-vegetated areas that lack actual structures, producing a cleaner building density map that better tracks economic activity.

4 Detailed Analysis

4.1 1. NDVI: The Clear Winner

The NDVI model (4 bands: NIR, Blue, SWIR2, NDVI) achieves the best district-level correlation despite having lower pixel AUC than the baseline (0.941 vs 0.976). This is the central finding of Stage 1c/2b:

Pixel AUC does not predict district-level economic correlation. The metrics optimize for different things — AUC measures within-patch discrimination, while district r measures whether spatial aggregation of predictions tracks real economic activity across 30 diverse districts.

The NDVI model likely succeeds because it produces fewer false positives in rural/agricultural districts (high NDVI = vegetation = not a building), while maintaining true positives in urban areas (low NDVI = built-up). This reduces the noise that decoupled building pixel counts from establishment counts in the baseline model.

4.2 2. S2-all: Good but Redundant

The 10-band S2 model (r = 0.727, 2023) performs almost as well as NDVI (r = 0.750). The full S2 stack includes NDVI-correlated bands (Red, Red Edge) that implicitly encode vegetation information. However, the additional 6 bands beyond the NDVI model’s 4 don’t help — they add noise without new discriminative information for this task.

Recommendation: Use the 4-band NDVI model. It achieves 97% of the 10-band model’s district correlation with 60% fewer input bands, meaning faster inference, smaller data footprint, and simpler preprocessing.

4.3 3. AE-only: Catastrophic Failure

The Alpha Earth model produces negative correlations (r = -0.289 for 2020, r = -0.279 for 2023). This means districts with more predicted buildings have fewer actual establishments — the model’s spatial pattern is inverted relative to economic reality.

Root cause: Inspection of the AE-only predictions reveals that 11 out of 30 districts have zero building pixels. The model, operating on 64 learned embedding dimensions with a threshold of 0.05, fails to produce meaningful spatial variation. It detects “buildings” in a pattern that reflects Alpha Earth’s internal feature space (likely correlated with land cover type) rather than actual built-up areas. Districts with certain land cover profiles (e.g., dense agriculture) get high predictions while genuinely urban districts get low predictions.

This is not a training failure — the model converged (AUC = 0.921 on the validation set). The problem is that Alpha Earth embeddings encode land semantics at landscape scale, not building presence at structure scale. A 10m embedding that says “this area is agricultural” provides no information about whether a specific pixel contains a building.

4.4 4. S2+AE: AE Branch Poisons the Model

The dual-branch model produces negative correlations (r = -0.273 for 2020, r = -0.260 for 2023) — nearly identical to AE-only. Despite having a dedicated S2 encoder branch with the same spectral bands that achieve r = 0.73 in the S2-all model, the AE branch dominates the fused representation and imposes its inverted spatial pattern.

Multiple districts show 0% building pixels (Bugesera, Gisagara, Rwamagana, Kirehe, Kayonza, and notably Kicukiro — a Kigali district with 14,883 establishments). The dual-branch architecture does not protect the S2 signal from AE contamination at the fusion layer.

Conclusion: Alpha Earth embeddings are not just unhelpful — they actively degrade building detection when combined with S2 bands. The learned features encode spatial patterns that are anti-correlated with economic activity at district scale.

5 The Key Insight: Pixel AUC vs. District Correlation

The results reveal a striking disconnect:

Pixel AUC ranking:     NIR (0.976) > NDVI (0.941) > S2-all (0.938) > S2+AE (0.931) > AE (0.921)
District r ranking:    NDVI (0.750) > S2-all (0.727) > NIR (0.432) > AE (-0.279)

The rankings are nearly inverted. The model with the best pixel-level discrimination produces the worst district-level economic signal (excluding AE’s catastrophic failure). Why?

  1. Pixel AUC rewards confidence on the validation set. The NIR model is extremely confident (threshold 0.90) — it knows exactly which patches in the validation set are buildings. But this confidence doesn’t account for false positive patterns that vary systematically across districts.

  2. District r rewards spatial consistency. What matters at district level is not whether each pixel is correct, but whether the aggregate building count per district is proportional to actual economic activity. The NDVI model makes more pixel-level errors (lower AUC) but makes them uniformly across districts, so the spatial pattern of aggregated predictions still tracks the census.

  3. The NIR model’s false positives are geographically biased. Without NDVI, the model cannot distinguish bare soil from built-up areas. Arid eastern districts (Nyagatare, Bugesera) get inflated building counts from bare soil false positives, while forested western districts get suppressed counts. This geographic bias decorrelates predictions from the census.

This has a direct methodological implication: always evaluate satellite-derived economic proxies at the unit of economic analysis (district), not at pixel level. Pixel metrics are necessary for debugging but insufficient for validation.

6 Recommendations

6.1 Immediate

  1. Adopt the NDVI model as the production model for Rwanda. It achieves r = 0.75 at district level with only 4 input bands and the simplest preprocessing pipeline.

  2. Retire the AE-only approach. Alpha Earth embeddings do not encode building presence at 10m resolution. They may be useful for land cover classification but not for structure-level detection.

  3. Drop S2+AE. The dual-branch model (r = -0.27) performs as badly as AE-only — the AE branch overwhelms the S2 branch at fusion.

6.2 For Stage 3 (GDP Regression)

The r = 0.75 result means building density from the NDVI model explains ~56% of the variance in district-level establishment counts. This is a strong enough economic signal to serve as a feature in a GDP regression model. When Peru fine-GDP data becomes available, the NDVI building density map can be used as:

  • An input feature to a regression model predicting GDP per pixel
  • A stratification layer for sampling (dense vs. sparse built-up areas)
  • A spatial prior for the regression head (areas with no buildings should have near-zero GDP)

6.3 For Global Application

The 4-band NDVI model requires only NIR, Blue, SWIR2, and NDVI — all derivable from Sentinel-2, which is globally available and free. No dependency on Alpha Earth or other commercial/restricted datasets. This makes the NDVI model the most scalable option for the eventual global GDP map (Stage 4).
